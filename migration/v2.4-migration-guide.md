# **Fugue v2.4 Migration Guide**  
**Status:** Authoritative  
**Governance Authority:** Curator  
**Scope:** Migration from v2.3 → v2.4  
**Purpose:** Define deterministic, governance‑aligned steps to uplift all governed artefacts, structures, and contracts to v2.4.

---

# 1. Migration Overview

The v2.4 uplift introduces:

- hierarchy‑complete governance (Project → Branch → Theme → Tranche)  
- hierarchy‑complete DECOR  
- hierarchy‑complete Ticket Maps  
- hierarchy‑complete Replay Traces  
- hierarchy‑complete Determinism Envelope  
- hierarchy‑complete Identity Propagation  
- hierarchy‑complete Namespace Scheme  
- hierarchy‑complete Documentation Envelope  
- updated Lifecycle Contract  
- updated Governance Envelope  
- updated persona routing  
- updated naming rules  
- updated folder structures  
- updated bootstrap rules  

All v2.3 artefacts MUST be migrated to the v2.4 canonical structure.

---

# 2. Migration Phases

Migration MUST follow the deterministic sequence:

```
1. Discovery
2. Structural Uplift
3. Contract Uplift
4. Hierarchy Uplift
5. Artefact Uplift
6. Ticket Map Uplift
7. Determinism Uplift
8. Identity Uplift
9. Documentation Uplift
10. Verification
11. Reconciliation
12. Freeze
```

No phase may be skipped, merged, or reordered.

---

# 3. Phase 1 — Discovery

The Orchestrator MUST:

- enumerate all existing v2.3 artefacts  
- classify artefacts by type (DECOR, Ticket Map, documentation, logs, replay traces)  
- classify artefacts by tranche  
- identify missing artefacts  
- identify naming drift  
- identify namespace drift  
- identify identity drift  
- identify determinism drift  
- identify documentation drift  

Outputs:

- discovery-log.md  
- drift-report.md

# 15. Post‑Migration Requirements

After freeze:

- all new tranches MUST use v2.4  
- all new branches MUST use v2.4  
- all new themes MUST use v2.4  
- all new projects MUST use v2.4  
- v2.3 MUST NOT be used for new work  
- v2.3 artefacts MUST NOT be modified  

---

# 16. Completion Criteria

Migration is complete when:

- all structural contracts are v2.4  
- all hierarchy contracts are v2.4  
- all execution contracts are v2.4  
- all runtime contracts are v2.4  
- all artefacts are in v2.4 namespaces  
- all metadata is v2.4‑aligned  
- all identity lineage is v2.4‑aligned  
- all determinism surfaces are v2.4‑aligned  
- all documentation is v2.4‑aligned  
- all Ticket Maps are v2.4‑aligned  
- all replay traces are v2.4‑aligned  
- all drift is reconciled  
- Curator signs off freeze
