# **2. Governance Hierarchy (Structural Inheritance)**

```
Project
  ↓ inherits governance, DECOR, naming, namespace, identity, determinism
Branch
  ↓ inherits governance, DECOR, naming, namespace, identity, determinism
Theme
  ↓ inherits governance, DECOR, naming, namespace, identity, determinism
Tranche
  ↓ produces governed execution artefacts
Ticket
```

Each layer has:

- its own DECOR  
- its own Ticket Map  
- its own bootstrap lifecycle  
- its own governed artefacts  
- its own metadata surfaces

# **4. DECOR (Declarative Contract of Record)**

DECOR governs:

- architectural structure  
- governance constraints  
- naming  
- namespace  
- identity lineage  
- metadata invariants  
- persona routing  
- documentation surfaces  
- determinism surfaces  

DECOR exists at:

- Project  
- Branch  
- Theme  
- Tranche  

DECOR is inherited downward and reconciled upward.

---

# **5. Ticket Map (Deterministic Execution Plan)**

Ticket Maps exist at:

- Project  
- Branch  
- Theme  
- Tranche  

Ticket Maps define:

- deterministic ticket decomposition  
- sequencing  
- dependencies  
- persona routing  
- DECOR subsets  
- naming + namespace metadata  
- identity metadata  
- determinism metadata  
- documentation metadata  

Ticket Maps MUST be deterministic, reproducible, and hierarchy‑aligned.

---

# **6. Replay Trace (Deterministic Execution Record)**

Replay traces exist at:

- Project  
- Branch  
- Theme  
- Tranche  

Replay traces MUST include:

- identity lineage  
- namespace lineage  
- evaluator metadata  
- evaluator diagnostics  
- implementation steps  
- bootstrap steps  
- state transitions  
- determinism metadata  

Replay traces are immutable after reconciliation.

# **8. Metadata Surfaces**

All governed artefacts MUST include:

- naming metadata  
- namespace metadata  
- identity metadata  
- determinism metadata  
- documentation metadata  
- lifecycle metadata  
- hierarchy metadata  

Metadata MUST NOT be mutated implicitly.

---

# **9. Drift Surfaces**

Drift MUST be classified as:

- **Critical** (structural, identity, determinism, lifecycle)  
- **Major** (ordering, metadata, documentation)  
- **Minor** (formatting, non‑blocking metadata)  

All drift MUST be reconciled via governed processes.
