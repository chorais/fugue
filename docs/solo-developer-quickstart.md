# **Solo Developer Quick‑Start Guide**  
Version: 2.4
### *How to use Fugue to build large, complex projects on your own — safely, predictably, and without losing context*

This guide gives you the **minimum you need** to start using Fugue as a solo developer.  
It focuses on:

- predictable flows  
- low cognitive load  
- safe large‑scale development  
- long‑term continuity  
- restart/resume capability  

---

# **1. Start a New Tranche**

### **Step 1 — Seed the Initiator**
Tell the Initiator:

> “I want to start a new tranche. Here is the mission.”

The Initiator produces the **Preamble**.

---

### **Step 2 — Seed the Orchestrator**
Attach the Preamble and say:

> “Bootstrap this tranche.”

The Orchestrator produces:

- Ticket Map  
- Ticket Map and implementation tickets  
- DECOR deltas (if needed)

---

### **Step 3 — Seed the Architect**
Attach the Preamble and say:

> “Produce the Initial DECOR (decor.md)."

This becomes the architectural contract.

---

# **2. Execute the Work**

### **Step 4 — Implement Tickets**
For each ticket:

- Start a fresh Implementer conversation  
- Attach the ticket + DECOR subset  
- Say:  
  > “Execute this ticket.”  

The Implementer produces:

- code  
- logs  
- traces  

---

# **3. Verify and Close the Tranche**

### **Step 5 — Seed the Auditor**
Attach:

- Initial DECOR (decor.md)  
- Ticket Map  
- Ticket Map and implementation tickets  
- Logs  
- Traces  

Say:

> “Perform reconciliation for this tranche.”

The Auditor produces:

- Reconciled DECOR  
- Closure Report  

This completes the tranche.

---

# **4. Restarting After Time Away (New Tranche)**

If you return after days/weeks/months:

### Use the **Restart Seeding Guide**  
Attach:

- Preamble  
- Reconciled DECOR  
- Closure Report  

Say:

> “Reconstruct the project state and propose the next tranche.”

This is governed by the **Restart Continuity Contract (RCC)**.

---

# **5. Resuming a Paused Tranche (Mid‑Execution)**

If you paused mid‑tranche:

### Use the **Tranche Resume Pattern (TRP)**  
Attach:

- Ticket Map  
- Ticket Map and implementation tickets  
- Last executed ticket  
- Implementer output  

Say:

> “Reconstruct the tranche state and identify the next ticket.”

This restores execution continuity.

---

# **6. When You Get Stuck**

If you’re unsure what to do next:

### Ask the Orchestrator:

> “What is the next logical step in the lifecycle?”

Or:

> “Summarise the current state of the project.”

The Orchestrator always knows the next move.

---

# **7. What You Get as a Solo Developer**

Fugue gives you:

- **a team’s worth of structure**  
- **predictable flows**  
- **safe large‑scale development**  
- **zero re‑derivation restarts**  
- **mid‑execution resume capability**  
- **durable artefacts that preserve intent**  
- **the ability to build systems normally requiring multiple engineers**  

Fugue supports solo developers by providing the structure and continuity needed to tackle larger or longer‑running projects, especially when working with AI‑assisted development tools.
